## Design Context

### Users
Security professionals, CISOs, and red team consultants reviewing incident response simulation results. They're reading this after a 25-120 minute pipeline run. They need to extract findings, share with leadership, and plan remediation. They're used to reading pentest reports, SIEM dashboards, and compliance docs.

### Brand Personality
Technical. Precise. Trustworthy.

### Aesthetic Direction
- **Visual tone:** Pentest report brought to life. Structured, evidence-heavy, monospace throughout. Not a marketing dashboard, not a SaaS app. A technical instrument for security professionals.
- **References:** Pentest report PDFs (structured findings with evidence), Burp Suite reports, NIST compliance documents. The credibility comes from structure and precision, not visual flair.
- **Anti-references:** Generic SaaS dashboards with big hero numbers and gradient cards. Loose, airy marketing layouts. Anything that looks like it prioritizes aesthetics over information.
- **Theme:** Light mode only. Warm neutral background (OKLCH pitch-black-50). White cards with subtle corner brackets (┌┐└┘). Monospace fonts throughout (Geist Mono body, JetBrains Mono code).
- **Color:** OKLCH warm palette. Royal Azure (primary/interactive), Verdigris (success/contained), Tuscan Sun (warning), Burnt Peach (destructive/critical). Color is semantic, never decorative.

### Design Principles
1. **Every pixel is evidence.** Dead space is wasted credibility. If a section has room, fill it with relevant data or remove the section.
2. **Horizontal before vertical.** Use 2-column layouts aggressively. Side-by-side comparisons, metrics beside narratives, actions beside risk. Reduce scrolling.
3. **Structure communicates trust.** Consistent section headers with sigils, dotted dividers between groups, ASCII progress bars for proportional data. The structure itself signals rigor.
4. **Density matches domain.** Security professionals expect dense, scannable information. Tight spacing within groups, clear separation between groups. No uniform padding everywhere.
5. **The report is the deliverable.** It should look good in a screenshot, in a demo video, and when exported to markdown. Every view should feel complete, not like a loading state.
